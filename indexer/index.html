    Indexer Documentation

    You can use BlockIndex to create your own custom indexers. This documentation will walk you through the process of creating a new indexer.

    How to create a new Indexer

    We take ERC20 as an example.

    To create a new indexer, you can simply POST /v1/api/indexer API with the following payload

    {
    "name": "test",
    "fields": {
      "contract_address": {
        "type": "string",
        "value": "{{ event['address'] }}"
      },
      "from": {
        "type": "string",
        "value": "{{ event['args']['from'] }}"
      },
      "to": {
        "type": "string",
        "value": "{{ event['args']['to'] }}"
      },
      "value": {
        "type": "int",
        "value": "{{ event['args']['value'] }}",
        "default": 0
      }
    },
    "abi": ["erc20 abi here..."],
    "trigger": {"event": "Transfer"},
    "chain_type": "ETH"
}

    The name field is the name of the indexer. The fields field is the fields that you want to index. The abi field is the abi of the contract. The trigger field is the event that you want to trigger the indexer. The chain_type field is the chain type that you want to index.

    The fields field is a dictionary. The key is the field name. The value is a dictionary. The properties of the dictionary are:

    • type: The type of the field. The type can be string, int

    • value: The value of the field. The value is a jinja2 template string. You can use the syntax {{ var }} to get access the value of variable var. There are some predefined variables that you can use:

      • event: The event that triggered the indexer. It contains all the information of the event.
      • event['address']: The address of the contract that triggered the indexer.
      • event['args']: The arguments of the event.
      • event['args']['from']: The from argument of the event.
      • event['args']['to']: The to argument of the event.
      • event['args']['value']: The value argument of the event.
      • event['blockNumber']: The block number of the event.
      • event['transactionHash']: The transaction hash of the event.
      • event['transactionIndex']: The transaction index of the event.

      • event['logIndex']: The log index of the event.

      • tx: The transaction that triggered the indexer. It contains all the information in the transaction.

      • tx['hash']: The transaction hash.
      • tx['nonce']: The nonce of the transaction.
      • tx['blockHash']: The block hash of the transaction.
      • tx['blockNumber']: The block number of the transaction.
      • tx['transactionIndex']: The transaction index of the transaction.
      • tx['from']: The from address of the transaction.
      • tx['to']: The to address of the transaction.
      • tx['value']: The value of the transaction.
      • tx['gas']: The gas of the transaction.
      • tx['gasPrice']: The gas price of the transaction.

      • tx['input']: The input of the transaction.

      • block: The block that triggered the indexer. It contains all the information in the block.

      • block['number']: The block number.
      • block['hash']: The block hash.
      • block['timestamp']: The timestamp of the block.
      • block['parentHash']: The parent hash of the block.
      • block['nonce']: The nonce of the block.

      • block['transactions']: The transactions root of the block.

      • timestamp: Current timestamp

    • default: The default value of the field.

    And then a table will be created in the database. The table name is {name}_indexer. The table will have columns based on the fields you defined. When indexing criteria is met, the indexer will be triggered. The indexer will insert a row into the table with the values you defined.

    Examples

    You can find examples of basic ERC20 and ERC721 transaction indexers under the examples/indexer_inputs folder.
